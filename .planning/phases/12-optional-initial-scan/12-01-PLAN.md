# Plan 12-01: Deltas-Only Mode (Skip Initial Scan)

**Wave:** 1
**Files modified:** `MenuBarManager.swift`, `MenuBarView.swift`, `SettingsStore.swift`

## Goal

Allow users to skip the initial full scan entirely. The app starts tracking changes immediately via FSEvents. Categories appear as changes are detected. A background reconciliation scan fills in total sizes later.

## Tasks

### Task 1: Add Tracking Mode to Onboarding

In the onboarding flow (MenuBarView `setupOnboardingView`):

- After folder selection (page 2), the scan page (page 3) offers two options:
  - "Scan now" - full scan, see everything immediately (current behavior)
  - "Start tracking" - skip scan, begin watching for changes immediately
- "Start tracking" creates the tracked path entries and enables FSEvents but does NOT trigger `loadInventory()`

### Task 2: Deltas-Only State in MenuBarManager

Add `isDeltasOnlyMode: Bool` computed property:

- True when enabled tracked paths exist but no snapshot has ever been taken
- When true, FSEvents is active and incremental updates work normally
- Categories only show growth data (`+340 MB Developer`) not total sizes
- The "total size" portion of `CategoryInventoryItem.currentSizeBytes` represents accumulated deltas since tracking started, not absolute disk usage

### Task 3: Adapt Category Display for Deltas-Only

In `CategoryGrowthListView` / `CategoryInventoryRow`:

- When in deltas-only mode, show categories with growth-only display
- Instead of "Developer — 45.2 GB" show "Developer — +1.2 GB since tracking"
- Growth indicators still work normally (journal-based)
- Subcategory drill-down shows files by delta size, not absolute size

### Task 4: Background Upgrade to Full Picture

When in deltas-only mode:

- After a configurable delay (e.g., 10 minutes after onboarding) or on next app launch, trigger `performSilentReconciliation()` (from Phase 10, Plan 04)
- When reconciliation completes, `isDeltasOnlyMode` becomes false
- Categories seamlessly gain their total sizes
- No user action required - automatic upgrade

### Task 5: Persist Tracking Start Timestamp

- Store `trackingStartedAt: Date` in UserDefaults when deltas-only mode begins
- Use this as the reference point for "since tracking" display
- Clear when first full scan completes

## Verification

- [ ] User can complete onboarding without any scan (instant)
- [ ] FSEvents starts immediately after "Start tracking"
- [ ] Categories appear as changes are detected
- [ ] Growth indicators show "+X MB since tracking"
- [ ] Background reconciliation fills in total sizes automatically
- [ ] After reconciliation, display switches to normal mode seamlessly
- [ ] Subcategory drill-down works in deltas-only mode
