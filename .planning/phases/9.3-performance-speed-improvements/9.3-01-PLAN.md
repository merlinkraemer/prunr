  <name>Task 1: Optimize batch inserts with GRDB prepared statements</name>
  <files>Prunr/Database/DatabaseManager.swift</files>
  <action>
Replace row-by-row insert loop (lines 157-164) with GRDB's executeBatch using prepared statement.

Current implementation:
```swift
for scanResult in batch {
    var entry = SnapshotEntry(...)
    try entry.insert(db)
}
```

Replace with:
```swift
let statement = try db.makeInsertStatement(sql: "INSERT INTO snapshotEntry (snapshotId, path, sizeBytes) VALUES (?, ?, ?)")
for scanResult in batch {
    try statement.execute(arguments: [snapshotId, scanResult.path, scanResult.sizeBytes])
}
```

This eliminates repeated SQL statement preparation overhead per row.

NOTE: Do NOT modify ScanService.swift batch size - keep the 2000-item batch there. Only optimize the database insert mechanism.
