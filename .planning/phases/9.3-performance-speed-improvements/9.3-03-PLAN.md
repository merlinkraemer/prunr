1. Simplify progress calculation (lines 193-221) from complex time-based分段 estimation to linear approximation:

Replace entire time-based分段 logic with:
```swift
let elapsed = now.timeIntervalSince(scanStartTime)
let percentage: Double
let estimatedTotal: Int

if elapsed < 2.0 {
    percentage = 0.0
    estimatedTotal = 0
} else {
    // Linear approximation: estimate based on current rate
    let rate = Double(count) / elapsed
    estimatedTotal = Int(rate * elapsed * 1.5) // Assume 50% more remaining
    percentage = min(0.95, Double(count) / Double(max(1, estimatedTotal)))
}
```

2. Reduce Task.yield() frequency from every batch (line 184) to every 2 batches for less coordination overhead:

Change from:
```swift
if batch.count >= batchSize {
    ...
    await Task.yield()
}
```

To:
```swift
if batch.count >= batchSize {
    ...
    if count % (batchSize * 2) == 0 {
        await Task.yield()
    }
}
```

This yields every 4000 items instead of every 2000, reducing overhead while maintaining responsiveness.
