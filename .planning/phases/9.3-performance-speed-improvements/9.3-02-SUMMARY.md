## Files Created/Modified

- `Prunr/Views/CategoryGrowthListView.swift`
  - Added `@State private var cachedSortedFolders` and `lastCachedCategory` for caching
  - Modified `sortedFolders(for:)` to return cached result when category unchanged
  - Added `.id()` modifiers to FolderHeaderRow, ItemRow, and NestedBigItemRow
  - Split complex chained closure into discrete steps (grouped → sorted items → sorted folders)

## Performance Impact

- **Before:** O(n log n) sorting executed on every view render (scroll, hover, any state change)
- **After:** O(n log n) sorting executed once per category, cached for all subsequent renders
- **Result:** Significantly reduced CPU usage during list interactions, smoother drill-down navigation

## Decisions Made

- Cache per category ID (`category.id`) rather than full object comparison - more efficient equality check
- Split complex chained closure into discrete steps - required to avoid Swift compiler timeout
- Keep helper methods (`itemsGroupedByFolder`, `sortedItems`) unchanged - maintains code clarity and separation of concerns
