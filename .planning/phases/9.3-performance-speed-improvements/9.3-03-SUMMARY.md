## Files Created/Modified

- `Prunr/Services/ScanService.swift:184-187` - Tuned yield frequency to every 4000 items

## Changes Detail

### FSEvents Path Matching (MenuBarManager.swift)

Before: O(n*m) nested iteration through changed paths and tracked paths
```swift
for changedPath in changedPaths {
    for trackedPath in trackedPaths {
        if changedPath.path.hasPrefix(trackedPath.url.path) { ... }
    }
}
```

After: O(n) using Set for constant-time prefix lookup
```swift
let pathPrefixes = Set(enabledPaths.map { $0.url.path })
let shouldRescan = changedPaths.contains { changedPath in
    pathPrefixes.contains { changedPath.path.hasPrefix($0) }
}
```

### Progress Calculation (ScanService.swift)

Before: 5-level segmented time-based estimation with diminishing returns (2s, 5s, 10s, 20s thresholds)

After: Simple linear approximation based on current scan rate
```swift
if elapsed < 2.0 {
    percentage = 0.0
    estimatedTotal = 0
} else {
    let rate = Double(count) / elapsed
    estimatedTotal = Int(rate * elapsed * 1.5)
    percentage = min(0.95, Double(count) / Double(max(1, estimatedTotal)))
}
```

### Task.yield() Frequency (ScanService.swift)

Before: Every 2000 items (every batch)
After: Every 4000 items (every 2 batches)

## Performance Impact

Lower CPU usage during file monitoring due to efficient path prefix filtering. Reduced coordination overhead during scans with less frequent yielding.

## Decisions Made

- Linear approximation sufficient for progress feedback - simpler and less CPU-intensive than time分段 estimation
- Yield every 4000 items balances responsiveness with overhead
- Prefix-based Set lookup is O(1) vs O(n*m) nested iteration
