The `sortedFolders(for:)` method (lines 168-177) is called on every view render and performs O(n log n) sorting.

Add a cached version computed only when category changes:

1. Add @State property after line 61:
```swift
@State private var cachedSortedFolders: [(path: String, items: [GrowthItem])] = []
@State private var lastCachedCategory: CategoryGrowthItem?
```

2. Replace `sortedFolders(for:)` method with caching version:
```swift
private func sortedFolders(for category: CategoryGrowthItem) -> [(path: String, items: [GrowthItem])] {
    // Return cached if category hasn't changed
    if let lastCached = lastCachedCategory, lastCached.id == category.id {
        return cachedSortedFolders
    }

    // Compute and cache
    let result = itemsGroupedByFolder(for: category)
        .map { (path, items) in
            (path, items.sorted { $0.growthBytes > $1.growthBytes })
        }
        .sorted {
            $0.items.reduce(0) { $0 + $1.growthBytes } >
            $1.items.reduce(0) { $0 + $1.growthBytes }
        }

    cachedSortedFolders = result
    lastCachedCategory = category
    return result
}
```

This ensures sorting happens once when entering drill-down, not on every render cycle.

NOTE: Keep `itemsGroupedByFolder(for:)` and `sortedItems(for:)` methods unchanged - they're helper methods.
