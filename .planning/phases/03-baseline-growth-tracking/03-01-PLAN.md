  <action>
Extend BaselineService with growth list calculation:

**Add GrowthItem struct (inside BaselineService file):**
```swift
struct GrowthItem: Identifiable {
    let id = UUID()
    let path: String
    let growthBytes: Int64
    let currentSizeBytes: Int64
    let percentOfParent: Double
}
```

**Add main method:**
```swift
func getGrowthList(trackedPath: TrackedPath) async throws -> [GrowthItem]
```

**Algorithm:**
1. Create a new "current" snapshot via `scanService.scan()`
2. Calculate deltas via `db.calculateDeltas(beforeId: baselineId, afterId: currentId)`
3. For each delta that grew (changeBytes > 0):
   - Calculate what percent of the delta's parent folder size this growth represents
   - Include in result if >= 70% threshold OR if it's a top-level item
4. Return sorted by growthBytes descending

**Drill-down helper:**
```swift
func drillDown(path: String, trackedPath: TrackedPath) async throws -> [GrowthItem]
```
- Same logic but filtered to children of the given path
- Stops at boundary folders using `boundaryConfig.shouldStopDrillDown(at: url)`

**What to avoid:**
- Don't drill into boundary folders (node_modules, .git, etc.) - use BoundaryConfig
- Don't include items that shrank or were deleted in the growth list
- Don't fetch all entries - use SQL filtering where possible
