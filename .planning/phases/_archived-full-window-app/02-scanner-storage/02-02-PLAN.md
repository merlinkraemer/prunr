<task type="auto">
  <name>Task 2: Create ScanService orchestrator</name>
  <files>Prunr/Services/ScanService.swift</files>
  <action>Create ScanService as an actor (for thread-safe state):

**Properties:**
- `static let shared = ScanService()` - singleton pattern
- `private let scanner = FileScanner()` - scanner actor
- `private let db = DatabaseManager.shared` - database reference

**Progress type:**
- `struct ScanProgress: Sendable` with `currentPath: String`, `foldersScanned: Int`, `currentSnapshotId: Int64?`

**Main scan method:**
- `func scan(path: String, progress: ((ScanProgress) -> Void)?) async throws -> Snapshot`

**Flow:**
1. Convert path String to URL(fileURLWithPath:)
2. Validate URL exists and is directory - throw ScanError.invalidPath if not
3. Create Snapshot via `await db.createSnapshot()`
4. Initialize counter and buffer
5. For-await loop over `await scanner.scan(url)`:
   - Accumulate ScanResults into batch array (max 2000)
   - When batch full: `await db.addEntries(to: snapshot.id, entries: batch)`
   - Clear batch, call `await Task.yield()`
   - Call progress callback with ScanProgress
6. After loop ends, insert any remaining partial batch
7. Return completed Snapshot

**Error handling:**
- Wrap errors in ScanError.unknown(originalError)
- File error → invalidPath, permission error → permissionDenied

**State management:**
- Add `@MainActor var isScanning = false` to track scan state
- Set isScanning = true at start, false in defer block
- Throw error if scan called while already scanning

**CRITICAL:** Use batch size of 2000 per research (sweet spot between 1000-5000). Always await Task.yield() between batches.</action>
