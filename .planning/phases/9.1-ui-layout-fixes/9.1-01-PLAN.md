# Plan: Layout & Header Fixes

## Objective

Fix column layout overflow, left-align titles/names, and update header displays to show proper information.

## Execution Context

**Critical files:**
- `Prunr/Views/CategoryGrowthListView.swift` - Column layout and alignment
- `Prunr/Views/MenuBarView.swift` - Monitor path header, bar header
- `Prunr/Views/DriveBarView.swift` - Free space display format

**Reference:**
- Current popup width: 320pt
- Current CategoryListRow total width: ~366pt (OVERFLOW)

## Context

### Root Cause (ISS-057)
The `CategoryListRow` uses fixed widths that exceed popup width:
- Icon: 20pt + Name: 120pt fixed + Count: 70pt + Size: 90pt + padding = ~366pt
- Popup is only 320pt wide, causing horizontal overflow

### Current Issues
1. **Column overflow:** Fixed 120pt name width causes content wider than window
2. **Alignment:** Names/titles not consistently left-aligned
3. **Monitor path header:** Shows redundant GB indicator
4. **Bar header:** Shows only free space, not total drive space

## Tasks

### Task 1: Fix Column Overflow (ISS-057)
**File:** `CategoryGrowthListView.swift`

Replace fixed-width name columns with flexible layout:

```swift
// BEFORE (CategoryListRow)
Text(item.category.displayName)
    .frame(width: 120)  // FIXED - causes overflow

// AFTER
Text(item.category.displayName)
    .frame(maxWidth: .infinity, alignment: .leading)  // FLEXIBLE
```

Apply to all row types:
- `CategoryListRow` - category name
- `FolderHeaderRow` - folder name
- `ItemRow` - file name
- `NestedBigItemRow` - file name

Remove or reduce `ColumnWidths.name` constant.

### Task 2: Left-Align All Titles/Names (ISS-058)
**File:** `CategoryGrowthListView.swift`

Ensure all name/title columns use `.alignment: .leading`:

```swift
Text(name)
    .lineLimit(1)
    .frame(maxWidth: .infinity, alignment: .leading)
```

Verify alignment in:
- Category names in main list
- Folder names in drill-down headers
- File names in detail rows
- Nested big file names

### Task 3: Remove GB Indicator from Monitor Path Header
**File:** `MenuBarView.swift`

Remove the size badge from the monitored path row. The path should show only the path name, not the size.

Find and remove the size display next to the monitor path (the "X GB" badge).

### Task 4: Update Bar Header Format
**File:** `DriveBarView.swift`

Change free space display from "X GB free" to "X GB free of Y GB":

```swift
// BEFORE
Text("\(freeGB) GB free")

// AFTER
Text("\(freeGB) GB free of \(totalGB) GB")
```

Use same formatting approach (1 decimal place for GB values).

## Verification

**Manual verification:**

- [ ] All names/titles are left-aligned

## Success Criteria

- No horizontal overflow in category or drill-down views
- Consistent left alignment across all row types
- Clean monitor path header without redundant size info
- Drive bar shows both free and total space

## Output

- Updated `CategoryGrowthListView.swift` with flexible column layout
- Updated `MenuBarView.swift` with cleaned header
- Updated `DriveBarView.swift` with new format
- All issues ISS-057, ISS-058 addressed
