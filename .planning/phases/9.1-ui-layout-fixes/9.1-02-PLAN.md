# Plan: Footer & Big File Visual Overhaul

## Objective

Redesign the footer as a horizontal icon-only toolbar and improve the visual treatment of nested big files in the category view.

## Execution Context

**Critical files:**
- `Prunr/Views/MenuBarView.swift` - Footer buttons
- `Prunr/Views/CategoryGrowthListView.swift` - NestedBigItemRow

**Design reference:**
```
Footer (horizontal icon-only):
┌─────────────────────────────────────┐
│  [🔍]    [↻]    [⚙]                │
│  Scan   Reset  Settings            │
└─────────────────────────────────────┘

Tooltips on hover, no visible text labels.
```

## Context

### Current Footer Issues (ISS-059)
- Vertical layout with text labels takes too much space
- Doesn't match macOS 26 minimalist patterns
- "Scan Now", "Reset Baseline", "Settings..." buttons are verbose

### Current Nested Big Files Issues (ISS-060)
- Tree characters (└── and ├──) look dated
- Visual hierarchy could be improved
- Indentation and spacing need refinement

## Tasks

### Task 1: Redesign Footer as Horizontal Icon-Only Toolbar (ISS-059)
**File:** `MenuBarView.swift`

Replace vertical button stack with horizontal icon toolbar:

```swift
// Footer toolbar
HStack(spacing: 24) {
    // Scan button
    Button(action: { /* scan */ }) {
        Image(systemName: "magnifyingglass")
            .font(.system(size: 14))
    }
    .buttonStyle(.plain)
    .help("Scan Now")

    // Reset baseline button
    Button(action: { /* reset */ }) {
        Image(systemName: "arrow.clockwise")
            .font(.system(size: 14))
    }
    .buttonStyle(.plain)
    .help("Reset Baseline")

    // Settings button
    Button(action: { /* settings */ }) {
        Image(systemName: "gearshape")
            .font(.system(size: 14))
    }
    .buttonStyle(.plain)
    .help("Settings")
}
.padding(.vertical, 8)
```

**Requirements:**
- Icons only, no text labels
- Use `.help()` modifier for tooltips
- Subtle hover state (opacity change or background)
- Centered horizontally in footer area
- Appropriate icon sizing (14-16pt)

### Task 2: Redesign Nested Big Files Visual Treatment (ISS-060)
**File:** `CategoryGrowthListView.swift`

Improve `NestedBigItemRow` visual design:

**Option A: Indented cards**
```swift
HStack(spacing: 8) {
    // Indent spacer
    Spacer().frame(width: 24)

    // File icon
    Image(systemName: "doc.fill")
        .foregroundStyle(.orange)

    // File name (flexible)
    Text(fileName)
        .frame(maxWidth: .infinity, alignment: .leading)

    // Size
    Text(sizeText)
        .foregroundStyle(.secondary)
}
.padding(.horizontal, 12)
.padding(.vertical, 4)
.background(Color.orange.opacity(0.05))
.cornerRadius(4)
```

**Option B: Simple indent with subtle connector**
- Remove tree characters (└── ├──)
- Use simple left indent (24-32pt)
- Subtle left border or dot indicator
- Orange tint for "big file" items

**Apply to:**
- `NestedBigItemRow` - nested big files under categories
- `MoreIndicatorRow` - "X more" indicator
