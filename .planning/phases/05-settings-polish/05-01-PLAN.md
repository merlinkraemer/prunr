<task type="auto">
  <name>Task 5: Add nested growth aggregation to BaselineService</name>
  <files>Prunr/Services/BaselineService.swift</files>
  <action>
Fix "No Changes" bug by aggregating nested file growth:

1. In getGrowthList, after fetching changes:
   - For each parent directory, sum growth of all descendants
   - If total nested growth > 0, create/merge GrowthItem for parent

2. Algorithm:
   ```
   For each GrowthItem in changes:
       For each ancestor path (parent, grandparent, etc.):
           ancestorGrowth[ancestor] += item.sizeChange
           ancestorFileCount[ancestor] += item.newFileCount

   For each ancestor with growth > 0:
       Create or update GrowthItem for ancestor
   ```

3. This ensures that changes in deep subfolders propagate up
4. Prevents "No Changes" when deep files grow but parent doesn't
  </action>
</task>
