<task type="auto">
  <name>Task 1: Configure window and menu commands</name>
  <files>Prunr/PrunrApp.swift, Prunr/Views/MainView.swift</files>
  <action>
Update PrunrApp.swift:
- Set default window size: .defaultSize(width: 800, height: 600)
- Set minimum size: .windowResizability(.contentMinSize)
- Add .commands modifier with custom menu items:
  - File menu: "Scan Home Folder" (Cmd+R) triggers scan
  - View menu: "Refresh" (Cmd+Shift+R) reloads snapshots

Update MainView if needed to expose scan action for menu command binding.

Use @Environment(\.openWindow) if needed for window management.
Keep menu items simple - just scan and refresh for v1.
  </action>
  <verify>
1. xcodegen generate && xcodebuild -project Prunr.xcodeproj -scheme Prunr build
2. Run app, verify window opens at reasonable size
3. Check File menu has "Scan Home Folder" with Cmd+R shortcut
  </verify>
  <done>Window has proper default size, menu commands work with keyboard shortcuts</done>
</task>

<task type="auto">
  <name>Task 2: Configure app identity and Info.plist</name>
  <files>project.yml</files>
  <action>
Update project.yml settings section:
- PRODUCT_BUNDLE_IDENTIFIER: com.yourname.Prunr (or appropriate identifier)
- MARKETING_VERSION: 1.0.0
- CURRENT_PROJECT_VERSION: 1
- INFOPLIST_KEY_LSApplicationCategoryType: public.app-category.utilities
- INFOPLIST_KEY_NSHumanReadableCopyright: Copyright 2026
- CODE_SIGN_STYLE: Automatic (or Manual if needed)
- DEVELOPMENT_TEAM: $(DEVELOPMENT_TEAM) - leave for user to set

Add Info.plist keys via project.yml:
- NSAppleEventsUsageDescription: Not used (placeholder for future)
- LSMinimumSystemVersion: 14.0

Regenerate project: xcodegen generate

Note: App icon (Assets.xcassets/AppIcon) is deferred - use default Xcode icon for v1.
  </action>
  <verify>
1. xcodegen generate succeeds
2. Open Prunr.xcodeproj in Xcode, check General tab shows correct bundle ID and version
3. Build succeeds
  </verify>
  <done>App has proper bundle identifier, version 1.0.0, and utility category</done>
</task>
