<task type="auto">
  <name>Task 1: Fix back button navigation (ISS-045)</name>
  <files>Prunr/Views/CategoryGrowthListView.swift</files>
  <action>
The back button in MenuBarView's drillDownHeader sets `manager.isDrilledDown = false` and `manager.selectedCategoryForDrilldown = nil`, but CategoryGrowthListView has internal `@State selectedCategory` that isn't reset.

The `computedSelectedCategory` property (line 22-24) uses `forcedCategory ?? selectedCategory`. Since no forcedCategory is passed (line 391 in MenuBarView), the internal `selectedCategory` stays set even after back is clicked.

**Fix:** Add `.onChange(of: manager.isDrilledDown)` modifier to reset internal `selectedCategory` when drill-down state changes from true to false:

```swift
// In CategoryGrowthListView body, after the ZStack's .animation modifier:
.onChange(of: manager.isDrilledDown) { _, newValue in
    if !newValue {
        // External back button was pressed, reset internal selection
        withAnimation(.easeInOut(duration: 0.3)) {
            selectedCategory = nil
        }
    }
}
```

This ensures that when MenuBarView's back button sets `manager.isDrilledDown = false`, CategoryGrowthListView responds by clearing its internal `selectedCategory`, allowing the view to transition back to the category list.
  </action>

  </verify>
  <done>
Back button in drill-down header navigates back to category list. View transitions properly with animation. No blank screen or stuck state.
  </done>
</task>

This ensures both headers have identical 20pt padding all around, creating visual consistency during navigation transitions.

<task type="auto">
  <name>Task 3: Fix footer separator spacing (ISS-047)</name>
  <files>Prunr/Views/MenuBarView.swift</files>
  <action>
In `mainCategoryView` (lines 136-151), there's a `Spacer()` between `pageNavigationContent` and the footer `Divider()`:

```swift
pageNavigationContent
Spacer()  // <-- Creates unwanted gap above footer
Divider()
footerButtons
```

This Spacer pushes footer to bottom but creates empty space when content doesn't fill the area.

**Fix:** Remove the `Spacer()` and instead make `pageNavigationContent` expand to fill available space by adding `.frame(maxHeight: .infinity)` to the VStack inside `pageNavigationContent`:

1. Remove `Spacer()` line between `pageNavigationContent` and `Divider()` (around line 145)
2. In `pageNavigationContent` (lines 167-175), wrap the VStack content in a container with `.frame(maxHeight: .infinity, alignment: .top)`:

```swift
private var pageNavigationContent: some View {
    VStack(spacing: 0) {
        headerSection
        Divider()
        categoryListView
    }
    .frame(maxHeight: .infinity, alignment: .top)
}
```

This makes the page content expand to fill available space while keeping content at the top, eliminating the gap above the footer separator.
  </action>

Build and run the app. The footer separator line should appear immediately below the category list content with no empty gap. Content should fill available space.
  </verify>
  <done>
No empty space between content and footer separator. Footer sits flush against the bottom of content. Clean, tight layout.
  </done>
