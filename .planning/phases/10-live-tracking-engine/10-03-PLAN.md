# Plan 10-03: Incremental Inventory Updates (No Cache Invalidation)

**Wave:** 2
**Depends on:** 10-01, 10-02
**Files modified:** `MenuBarManager.swift`, `RecentChangeService.swift`, `DatabaseManager.swift`

## Goal

When FSEvents reports changes and RecentChangeService updates the working set, patch only the affected category totals and subcategory data in-place. Stop invalidating all subcategory caches on every update. The UI should never show spinners after the initial scan.

## Tasks

### Task 1: Make RecentChangeService Return Category-Level Deltas

In `RecentChangeService.refreshChangedPaths()`:

- It already returns merged deltas as `[DatabaseManager.JournalDeltaKey: Int64]`
- Change `RefreshResult.updated` to carry the delta data: `.updated(deltas: [DatabaseManager.JournalDeltaKey: Int64])`
- This tells the caller exactly which categories changed and by how much

### Task 2: Patch Category Totals In-Place

In `MenuBarManager.performRecentChangeRefresh()` (~line 1989-2027):

- On `.updated(deltas)`: instead of calling `loadInventoryFromLatestSnapshot(invalidateSubcategoryCache: true)`, directly update the affected `CategoryInventoryItem` entries
- For each delta key, find the matching category in `growingCategories` or `stableCategories`
- Update `currentSizeBytes` by adding the delta
- If a category's growth journal now shows a trend, move it from `stableCategories` to `growingCategories` (and vice versa)
- This is O(number of changed categories), not O(all categories)

### Task 3: Selective Subcategory Cache Update

When a category is affected by an incremental update:

- Do NOT invalidate `subcategoryGroupsByCategory` for that category
- Instead, if the category has cached subcategory data, update the relevant subcategory's `totalBytes` and `fileCount` in-place
- Only invalidate a specific subcategory's `topFiles` if a file in that subcategory changed (the top-20 might have shifted)
- If the user is currently drilled into the affected subcategory, refresh just that view

### Task 4: Remove Blanket Cache Invalidation

In `applyInventory()` (~line 1263-1298):

- Remove `invalidateSubcategoryCache` parameter entirely
- `applyInventory` should only be called during initial scan / manual rescan now
- For incremental updates, use the new in-place patching from Task 2

In `loadInventoryFromLatestSnapshot()`:

- Remove the `invalidateSubcategoryCache: true` call from `performRecentChangeRefresh`
- This method is still used on app launch to load from DB, but not for live updates

### Task 5: Simplify Loading State Booleans

In `MenuBarManager.swift`:

- Remove `isAnalyzingChanges` - no longer needed (was for post-scan analysis phase)
- Remove `isCleaningUp` - DB cleanup can happen silently
- Remove `isProcessingRecentChanges` from driving any visible UI state - keep the property but don't pulse the menu bar icon for it
- Keep `isLoading` only for initial scan and manual rescan
- Keep `isAutoScanning` only for silent background reconciliation

In `MenuBarView.swift`:

- Simplify `footerStatusText`: show `lastDetectedChangeAt` relative time, or "No changes detected"
- Remove the "Scanning...", "Updating..." cycling text for background operations
- The `activityPulseTimer` / menu bar pulsing should only activate during manual scans, not incremental updates
