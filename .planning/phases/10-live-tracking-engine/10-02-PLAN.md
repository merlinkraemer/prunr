# Plan 10-02: Remove Auto-Rescan Machinery

**Wave:** 1
**Depends on:** none (can be done in parallel with 10-01)
**Files modified:** `MenuBarManager.swift`

## Goal

Remove the periodic full-rescan system. After initial baseline, the app should never automatically trigger a full scan. Only FSEvents-driven incremental updates and user-initiated manual rescans.

## Tasks

### Task 1: Remove Auto-Scan Timer Trigger

In `startRealtimeUpdates()` (line ~1633):

- Keep the timer for `updateFreeSpace()` (disk space display needs periodic refresh)
- Keep `configureFileWatcherIfNeeded()` call (ensures watcher stays running)
- Remove the `useFallbackAutoScan` check and `scheduleAutomaticScan` call
- Timer interval can stay at 15s for free space updates

### Task 2: Remove Auto-Scan Scheduling System

Delete these methods entirely from `MenuBarManager.swift`:

- `scheduleAutomaticScan(resetDebounce:)` (~line 2029-2072)
- `runAutomaticFullScan()` (~line 2119-2124)
- `runAutomaticFullScanIfPossible()` (~line 2112-2117)
- `automaticFullScanInterval(for:)` (~line 276-282)

Remove associated properties:

- `autoScanTask`
- `pendingFallbackAutoScanTick`
- `lastAutomaticScanAttemptAt`
- `useFallbackAutoScan`
- `normalAutoScanDebounce`, `pressureAutoScanDebounce`
- `normalAutoScanAttemptInterval`, `pressureAutoScanAttemptInterval`
- `startupAutoScanGracePeriod`

### Task 3: Remove Promote-to-Full-Scan from RecentChangeService

In `RecentChangeService.swift`:

- Remove `shouldPromoteToFullScan()` method entirely (~line 242-284)
- Remove the guard in `refreshChangedPaths()` that calls it (~line 55-57)
- Remove `maxIncrementalRefreshTargets`, `maxIncrementalRefreshSubtreeTargets`, `maxIncrementalRefreshFileTargets`, `maxIncrementalRefreshRemovalTargets` constants
- The `.needsFullScan` case in `RefreshResult` can stay but will never be returned

### Task 4: Remove Overflow Full Scan Handling

In `MenuBarManager.swift`:

- Remove `requestOverflowFullScan()` (~line 2103-2110)
- Remove `pendingOverflowFullScan` property
- Remove overflow handling in `endInventoryRefresh()` (~line 2094-2101)
- In the FSEvents `onChange` callback: remove the `changeBatch.requiresFullRescan` branch that calls `requestOverflowFullScan()`. Instead, just process the paths normally (large batches are fine now)

### Task 5: Keep Manual Scan Button Working

- `refreshVisibleInventory()` stays but rename to `manualRescan()` for clarity
- Keep the "Scan Now" / refresh button in the footer
- Keep `isLoading` for the manual scan loading state
- Keep `loadInventory()` as-is for manual triggering

## Verification

- [x] No automatic full scans occur after initial baseline
- [x] 15-second timer still updates free space display
- [x] Manual "Scan Now" button still triggers a full scan
- [x] Large FSEvents batches (>256 paths) are processed incrementally, not promoted to full scan
- [x] `isAutoScanning` is only true during manual rescan or reconciliation
