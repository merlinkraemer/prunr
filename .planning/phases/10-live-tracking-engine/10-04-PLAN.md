# Plan 10-04: Silent Background Reconciliation

**Wave:** 3
**Depends on:** 10-03
**Files modified:** `MenuBarManager.swift`

## Goal

Add a background reconciliation scan that silently corrects working set drift without any visible UI loading states. Runs on app launch (if stale) or on manual trigger. Applies corrections as incremental patches, not a full UI refresh.

## Tasks

### Task 1: Add Reconciliation Scan Method

Add `performSilentReconciliation()` to `MenuBarManager.swift`:

- Runs a full scan via `createBaselines()` in the background
- Does NOT set `isLoading = true` (invisible to user)
- Does NOT show scan progress, spinners, or status text
- After scan completes, compare new snapshot against current working set
- Apply differences as incremental category total patches (same as Task 2 in 10-03)
- Record any deltas to the growth journal
- Update `lastAutomaticScanAt` timestamp

### Task 2: Trigger on App Launch If Stale

In `MenuBarView.task` block (~line 321-337) or `autoInitializeIfNeeded()`:

- After loading cached inventory from latest snapshot, check `lastAutomaticScanAt`
- If nil or >24 hours ago, kick off `performSilentReconciliation()` as a detached task
- Do NOT await it - fire and forget, UI shows cached data immediately

### Task 3: Add Manual Reconciliation Trigger

- The existing "Scan Now" button in the footer triggers `refreshVisibleInventory()` which does a visible manual scan
- Keep this behavior but also update the reconciliation timestamp
- Optionally: add a "last full scan: 2h ago" indicator in the footer (subtle, non-spinner)

### Task 4: Track Reconciliation State Minimally

- Add `lastReconciliationAt: Date?` property
- Add `isReconciling: Bool` (internal only, does NOT drive any UI loading state)
- Guard against concurrent reconciliation + manual scan (if manual scan starts, cancel reconciliation)

## Verification

- [x] Reconciliation runs silently on app launch when last scan was >24h ago
- [x] No spinners, progress bars, or status text changes during reconciliation
- [x] Category totals update quietly when reconciliation finds drift
- [x] Subcategory caches are NOT invalidated during reconciliation
- [x] Manual "Scan Now" still works with visible progress (different code path)
- [x] Concurrent manual scan cancels in-progress reconciliation
