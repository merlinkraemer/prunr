# Plan 10-01: Enable FSEvents on All Paths + Noise Filter

**Wave:** 1
**Files modified:** `MenuBarManager.swift`, new `FSEventsNoiseFilter.swift`

## Goal

Make FSEvents the primary update engine for all tracked paths including home directory. Add a noise filter to suppress known-noisy paths that don't represent meaningful user-visible storage changes.

## Tasks

### Task 1: Create FSEventsNoiseFilter

Create `Prunr/Services/FSEventsNoiseFilter.swift`:

- A simple struct/enum with a static `shouldIgnore(_ path: String) -> Bool` method
- Filter patterns (skip stat, skip journal):
  - `.DS_Store`, `.localized`, `Thumbs.db`, `desktop.ini`
  - `Library/Saved Application State/`
  - `Library/Caches/com.apple.Safari/` (but NOT all caches - user cares about app caches growing)
  - `Library/Caches/CloudKit/`
  - `Library/Caches/com.apple.nsurlsessiond/`
  - `.Spotlight-V100/`
  - `.fseventsd/`
  - `.Trashes/` (system-level, not user .Trash)
  - `Library/WebKit/`
  - `Library/Metadata/`
  - Known IDE temp files: `.swp`, `.swo`, `~`, `.tmp`
  - Already-ignored names from `SettingsStore.allScanIgnoreNames`
- Use prefix matching for directory patterns, suffix matching for extensions
- Keep it fast - this runs on every FSEvent

### Task 2: Remove `shouldAutoWatchTrackedPath` Guard for Home Dir

In `MenuBarManager.swift`:

- Remove or modify `shouldAutoWatchTrackedPath()` (line ~266-274) which currently returns `false` for home dir and `/`
- For `/` (root): still return false (too noisy, not a realistic use case)
- For home dir: return true (this is the primary use case)

### Task 3: Watch All Enabled Tracked Paths

In `configureFileWatcherIfNeeded()` (line ~1919):

- Currently watches only the primary tracked path
- Change to collect URLs from ALL enabled tracked paths via `effectiveTrackedPaths()`
- Pass all URLs to a single `FSEventsWatcher` (FSEvents supports multiple paths)
- Apply `effectiveTrackedPaths` deduplication (already exists) to avoid watching nested paths

### Task 4: Apply Noise Filter in FSEvents Callback

In the FSEvents `onChange` callback (line ~1957-1971):

- Before adding to `pendingRecentChangePaths`, filter through `FSEventsNoiseFilter.shouldIgnore()`
- Filter at the event level, not after accumulation
