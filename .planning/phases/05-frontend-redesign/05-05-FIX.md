<task type="auto">
  <name>Task 3: Fix growth bars with minimum width and proper scaling</name>
  <files>Prunr/Views/GrowthBarView.swift</files>
  <action>
Current issue: All bars show at 100% width because maxChange equals each item's change (only one category, or single category dominates).

User decision: Option A (relative to largest) with minimum bar width

Changes:
1. Calculate bar width as proportion of largest change: width = change / maxChange
2. Apply minimum width of 20% so all bars are visible even when one dominates
3. Formula: minWidth + (width * (1 - minWidth)) where minWidth = 0.2
4. This ensures smallest bar is still visible (20%) while largest scales to 100%

Also fix color semantics:
- Growth (positive change) = RED (space was consumed)
- Shrinkage (negative change) = GREEN (space was freed)
- This matches the user's mental model: red = bad (more disk used)

Add clear labels showing absolute values next to bars.

User doesn't know which snapshots are being compared. Add a summary header.

Add to ScanResultsView (top of ScrollView):
1. Header showing: "Comparing {current time} vs {historical time}"
2. Times shown as relative (e.g., "2 hours ago") and absolute (e.g., "Jan 10, 2:30 PM")
3. Small, subtle text (caption/footnote style)
4. Shows which two snapshots were used

Properties needed in MainViewModel:

- currentSnapshotDate: Date?
- historicalSnapshotDate: Date?

Format:
"Comparing {formattedCurrent} vs {formattedHistorical}"
Where formatted is:
- If < 24h: "2 hours ago"
- If < 7d: "3 days ago"
- Otherwise: "Jan 10"

Add formatter in MainViewModel:
```swift
func formattedSnapshotDate(_ date: Date) -> String {
    let hoursAgo = Date().timeIntervalSince(date) / 3600
    if hoursAgo < 24 { return "\(Int(hoursAgo))h ago" }
    let daysAgo = Int(hoursAgo / 24)
    if daysAgo < 7 { return "\(daysAgo)d ago" }
    return DateFormatter.shortDate.string(from: date)
}
```
