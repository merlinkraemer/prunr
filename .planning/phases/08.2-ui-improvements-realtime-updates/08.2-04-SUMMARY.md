## Decisions Made

**1. forcedCategory parameter approach (ISS-043 fix)**
- Replaced boolean `isDetailView` flag with `CategoryGrowthItem?` parameter
- Rationale: Boolean flag caused blank screen when selectedCategory was nil but detail view was expected. Passing actual category ensures proper rendering.
- Pattern: `computedSelectedCategory` property combines forced and internal selection as single source of truth

**2. Remove background dimming overlay (ISS-036, ISS-040 fix)**
- Removed the dimming overlay that appeared during detail view
- Rationale: Overlay interfered with push animation perception and wasn't needed for proper navigation UX
- Result: Cleaner push animation with both views moving simultaneously

**3. True conditional rendering (ISS-036, ISS-040 fix)**
- Changed from ZStack with offsets to conditional rendering with `.transition(.asymmetric())`
- Rationale: Only ONE view exists at any time eliminates overlap and transparency fighting
- Implementation: `if computedSelectedCategory == nil { listView } else { detailView }`
- SwiftUI handles transition timing automatically

**4. Fixed modal size (ISS-044 fix)**
- Set scan modal to fixed 260x180 frame
- Rationale: Dynamic sizing caused jarring visual changes during progress updates
- Additional improvements: Increased spacing (16), padding (24), show progress at 1%
