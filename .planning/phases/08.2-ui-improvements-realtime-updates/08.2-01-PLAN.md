
New implementation:
```swift
var body: some View {
    ZStack {
        // Background dimming overlay (only in detail view)
        if selectedCategory != nil {
            Color.black.opacity(0.1)
                .ignoresSafeArea()
                .onTapGesture {
                    withAnimation(.easeInOut(duration: 0.3)) {
                        selectedCategory = nil
                        manager.isDrilledDown = false
                    }
                }
        }

        // Conditional rendering: ONLY ONE view exists at a time
        if selectedCategory == nil {
            // Main list view
            categoryListView
                .transition(.asymmetric(
                    insertion: .move(edge: .leading),
                    removal: .move(edge: .leading)
                ))
        } else {
            // Detail view
            categoryDetailView
                .transition(.asymmetric(
                    insertion: .move(edge: .trailing),
                    removal: .move(edge: .leading)
                ))
        }
    }
    .animation(.easeInOut(duration: 0.3), value: selectedCategory)
}
```

Key changes:
1. Removed .offset() modifiers - conditional rendering handles positioning
2. Added .transition(.asymmetric()) to both branches
3. List enters from leading (left), exits to leading (left) - appears to push left
4. Detail enters from trailing (right), exits to leading (left) - push effect
5. Back button naturally triggers reverse animation

Why this works: Only ONE view renders at a time (conditional if/else), eliminating overlap. SwiftUI handles transition timing automatically. No transparency fighting - views appear/disappear cleanly.
