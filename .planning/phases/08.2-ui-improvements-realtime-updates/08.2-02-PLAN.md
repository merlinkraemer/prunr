Add explicit call to updateMenuBarDisplay() after updating state:
```swift
func updateFreeSpace() {
    let free = DiskSpaceService.shared.getFreeSpace()
    let total = DiskSpaceService.shared.getTotalSpace()

    // Update observable state for UI (DriveBarView reacts automatically)
    self.freeBytes = free
    self.totalBytes = total
    self.usedBytes = total - free

    // CRITICAL: Explicitly sync AppKit menu bar (ISS-042)
    let gb = Double(free) / 1_000_000_000
    if gb >= 1000 {
        let tb = gb / 1000
        updateFreeSpaceDisplay("\(String(format: "%.1f", tb)) TB")
    } else {
        updateFreeSpaceDisplay("\(String(format: "%.1f", gb)) GB")
    }

    // Update cache timestamp
    lastFreeSpaceUpdate = Date()
}
```

Why this approach: updateFreeSpace() already updates observable state (freeBytes, totalBytes, usedBytes) which DriveBarView uses. We just ensure it's called after scans. The explicit updateMenuBarDisplay() call syncs NSStatusItem.button.title which @Observable doesn't handle.

2. Create startRealtimeUpdates() method:

Add after updateFreeSpaceDisplay() method (after line 603):
```swift
/// Starts continuous 2-second updates to keep menu bar in sync (ISS-042)
private func startRealtimeUpdates() {
    // Invalidate any existing timer
    updateTimer?.invalidate()

    // Create timer that fires every 2 seconds
    updateTimer = Timer.scheduledTimer(withTimeInterval: 2.0, repeats: true) { [weak self] _ in
        Task { @MainActor in
            self?.updateFreeSpace()
        }
    }

    print("[MenuBarManager] Started 2s realtime update timer")
}
```

3. Call startRealtimeUpdates() in init():

Find line 78 (in init after setupContextMenu()):
```swift
override init() {
    super.init()
    Self.shared = self
    setupMenuBar()
    setupContextMenu()
    updateFreeSpace()
    startWatchingDefaultPaths()

    // NEW: Start continuous updates (ISS-042)
    startRealtimeUpdates()
}
```

4. Add cleanup in deinit:

Add at end of class (before closing brace):
```swift
deinit {
    // Cleanup timer when manager is deallocated
    updateTimer?.invalidate()
    print("[MenuBarManager] Stopped realtime update timer")
}
```

Why 2 seconds: Matches cache interval from Phase 8.1 (balance between real-time feedback and performance). Timer fires every 2s, updateFreeSpace() respects cache, so actual disk checks happen at same 2s interval. This provides continuous sync without excessive disk I/O.

Why Timer.scheduledTimer: Simple, reliable approach. Alternative is Combine/Timer.publish but Timer is sufficient for this use case.

Why @MainActor: updateFreeSpace() updates @Observable properties which require main actor. Timer callback runs on background thread, so we dispatch to @MainActor.
