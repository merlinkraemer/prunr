  <files>Prunr/Views/MenuBarView.swift, Prunr/Views/DriveBarView.swift</files>
  <action>
    Enhance header section for better information hierarchy and clarity:

    **Current issues:**
    - Drive stats and path info have similar visual weight
    - Unclear separation between drive capacity and monitored path
    - Path size display could be more prominent
    - Loading state for path size calculation not clear

    **Improvements to implement:**

    **1. Strengthen drive bar visual prominence:**
    - Drive bar is primary info (most important) - keep current bold design
    - Ensure percentage badge is clearly visible
    - Verify color coding (green/orange/red) is working well

    **2. Clarify path section as secondary info:**
    - Add subtle section divider or spacing between drive bar and path
    - Consider light background tint for path row (very subtle, like 5% gray)
    - Or add 1pt separator line between sections

    **3. Improve path size display:**
    - Make path size badge more prominent (slightly larger font or more contrast)
    - Add loading indicator when calculating: "Calculating..." in place of size
    - Consider moving auto-scan indicator to this row if active

    **4. Better visual grouping:**
    - Increase spacing between drive bar and path info (from 12pt to 16pt)
    - Ensure alignment is clean (left-aligned icon+path, right-aligned size)
    - Consistent spacing and alignment throughout

    **5. Loading states:**
    - When updatePathSize() is running, show small spinner or "..." in size badge
    - Don't show "0 B" if size not calculated yet - show loading state

    **Implementation priorities:**
    - P0: Spacing between sections (16pt vs 12pt)
    - P0: Path size loading state
    - P1: Subtle separator or background tint
    - P2: Visual polish (alignment, contrast)

    **Design principle:**
    Drive bar = primary (bold, colored, prominent)
    Path info = secondary (subtle, gray, smaller)
    Clear hierarchy through size, weight, color
  </action>
