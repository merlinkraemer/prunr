<task type="auto">
  <name>Task 2: Optimize popup opening sequence</name>
  <files>Prunr/Services/MenuBarManager.swift, Prunr/Views/MenuBarView.swift</files>
  <action>
    Based on profiling results, optimize popup opening:

    **Likely optimizations:**

    **1. Defer expensive operations:**
    - Move updatePathSize() from .task {} to background Task after popup visible
    - Only run updateFreeSpace() if last update was >5s ago (cache result)
    - checkBaseline() is lightweight, keep it

    **2. Optimize MenuBarView.task:**
    ```swift
    .task {
        // Fast: Update disk space (cached if recent)
        manager.updateFreeSpace()

        // Fast: Check baseline exists (UserDefaults lookup)
        await manager.checkBaseline()

        // Slow: Defer to background after popup visible
        Task.detached(priority: .utility) {
            await manager.updatePathSize()
        }
    }
    ```

    **3. Lazy load category items:**
    - Don't load categoryItems on popup open
    - Show "Ready to scan" state initially
    - Only load on explicit "Scan Now" or auto-scan trigger
    - This avoids blocking popup open with scan operations

    **4. Optimize NSPopover show:**
    - Ensure popover.show() happens on main thread but doesn't block
    - Consider using CATransaction.setDisableActions(true) for instant display
    - Remove any animations that delay initial display

    **5. Cache frequently accessed data:**
    - Cache monitoredPathDisplay calculation
    - Cache drive bar calculations if values haven't changed
    - Use @Observable effectively to prevent redundant updates

    **Target: Popup opens in <100ms from click to visible**
  </action>
  <verify>Build in Release mode. Use Instruments to verify popup opening time is <100ms. Test 10 times, average should be <100ms with no outliers >150ms. Popup should feel instant when clicked.</verify>
