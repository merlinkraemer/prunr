  <name>Task 2: Implement minimum display duration for scanning indicators (ISS-022 Part 1)</name>
  <files>Prunr/Services/MenuBarManager.swift, Prunr/Views/MenuBarView.swift</files>
  <action>
    Fix quick scan flash by implementing minimum display duration:

    **Implementation approach:**
    1. Add scanStartTime: Date? property to MenuBarManager
    2. When setting isLoading = true or isAutoScanning = true, record Date()
    3. When scan completes, calculate elapsed time
    4. If elapsed < 800ms, add delay: try? await Task.sleep(for: .milliseconds(800 - elapsed))
    5. Then set isLoading = false / isAutoScanning = false

    **Apply to both:**
    - loadCategoryGrowthList() - manual scans
    - Auto-scan callback in startWatchingPaths() - background scans

    **Edge cases:**
    - If user clicks Stop during minimum display, honor cancellation immediately
    - If new scan starts during delay, cancel delay and start new scan
    - Don't apply minimum duration to baseline creation (already slow)

    **Constants:**
    - Minimum display duration: 800ms (long enough to perceive, short enough to not feel sluggish)
    - Only apply if scan completed successfully (not on errors or cancellation)
  </action>
