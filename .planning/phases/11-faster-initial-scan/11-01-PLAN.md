# Plan 11-01: Optimize DB Write Path

**Wave:** 1
**Files modified:** `DatabaseManager.swift`, `ScanService.swift`

## Goal

Fix the DB write bottleneck that causes 2.2M file scans to take 30+ minutes. The FTS traversal takes ~30s; the remaining time is DB overhead from per-row path ID lookups and classification upserts.

## Tasks

### Task 1: Bulk Path Classification Upserts

In `upsertPathClassifications()` (~line 1378):

- Currently loops individual `INSERT ... ON CONFLICT DO UPDATE` per path (2.2M individual statements)
- Change to multi-row VALUES like `fetchPathIds` already does for path inserts
- Chunk into batches of 500, build `INSERT INTO pathClassification (pathId, category, subcategory) VALUES (?,?,?), (?,?,?), ... ON CONFLICT(pathId) DO UPDATE SET ...`

### Task 2: Bulk Snapshot Entry Inserts

In `addEntries()` (~line 496):

- Currently uses prepared statement executed per row in a loop
- Change to multi-row INSERT: `INSERT INTO snapshotEntry (snapshotId, pathId, sizeBytes) VALUES (?,?,?), (?,?,?), ...`
- Chunk into groups of 500 values per statement

### Task 3: Remove COLLATE NOCASE from Path Lookups

In `fetchPathIds()` (~line 1446):

- `SELECT id, path FROM paths WHERE path IN (...) COLLATE NOCASE` forces case-insensitive comparison on every lookup
- macOS paths are case-preserving. Normalize to canonical form once on insert (already done via `normalizePath`)
- Remove `COLLATE NOCASE` - use exact match
- Also remove it from `getOrCreatePathId` (~line 1411)

### Task 4: Increase Batch Size

In `ScanService.scan()` (~line 206):

- Change `batchSize` from 5000 to 15000
- Fewer transactions = less overhead
- Adjust yield frequency accordingly (every 2 batches -> every batch at 15000)

### Task 5: Build Working Set During Scan

In `BaselineService.createBaseline()` (~line 72):

- Currently: scan -> `addEntries` to snapshotEntry -> `rebuildWorkingSet` copies 2.2M rows
- Change: have `addEntries` also write to `workingSetEntry` in the same transaction
- Add parameter `alsoWriteWorkingSet: Bool` to `addEntries` (or a new method `addEntriesWithWorkingSet`)
- Skip the separate `rebuildWorkingSet` call when this flag is true
- The working set category totals can be written once at the end from the in-memory `categoryTotals` dict
