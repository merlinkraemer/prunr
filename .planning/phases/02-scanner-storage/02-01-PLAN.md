# PLAN 02-01: Disk Scanner

## objective
Create a disk scanner service that recursively scans directories and calculates folder sizes using FileManager APIs.

## execution_context
@.planning/STATE.md
@.planning/ROADMAP.md
@Prunr/Models/Snapshot.swift
@Prunr/Database/DatabaseManager.swift

## context
**From Phase 1:**
- DatabaseManager.shared singleton pattern established
- Snapshot(id, createdAt) and SnapshotEntry(id, snapshotId, path, sizeBytes) models defined
- GRDB.swift integration complete with migrations

**This Plan:**
- Pure Swift FileManager usage (no new dependencies)
- Focus on scanning logic only; storage integration in 02-02
- Async/await for non-blocking operations
- Progress callback for UI updates during long scans

## tasks

### Task 1: Create DiskScanner class with recursive folder size calculation

**File to create:** `Prunr/Services/DiskScanner.swift`

**Requirements:**
- Class name: `DiskScanner`
- Method: `scanDirectory(at path: String) throws -> [FolderSize]`
- Struct: `FolderSize(path: String, sizeBytes: Int64)`
- Use `FileManager.default.enumerator(at:includingPropertiesForKeys:)` for traversal
- Use `URLResourceKey.fileSizeKey` to get folder sizes
- Skip hidden files (starting with `.`)
- Handle permission errors gracefully (log and continue)

**Verification:**
- Build compiles: `xcodebuild -scheme Prunr -sdk macos build`
- Manual: Can scan a test directory and get accurate folder sizes

---

### Task 2: Add async scanning with progress callback

**File to modify:** `Prunr/Services/DiskScanner.swift`

**Requirements:**
- Add async method: `func scanDirectory(at path: String) async throws -> [FolderSize]`
- Add progress callback: `typealias ProgressHandler = (currentPath: String, foldersScanned: Int) -> Void`
- Update scan method to accept optional progress handler
- Call progress handler for each folder scanned

**Verification:**
- Build compiles
- Manual: Progress callback updates during scan
