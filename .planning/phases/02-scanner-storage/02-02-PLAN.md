# PLAN 02-02: Snapshot Storage Integration

## objective
Integrate DiskScanner with DatabaseManager to store scan results, and create ScanService as the single entry point for scanning operations.

## context
**From Phase 1:**
- DatabaseManager.shared with write() and read() methods
- Snapshot and SnapshotEntry models with GRDB conformance
- Associations defined: Snapshot.entries, SnapshotEntry.snapshot

**From 02-01:**
- DiskScanner class with `scanDirectory(at:)` method returning `[FolderSize]`
- Async scanning with progress callback

**This Plan:**
- Add CRUD methods to DatabaseManager for snapshots and entries
- Create ScanService to orchestrate scanner + storage
- Single entry point: `ScanService.shared.scan(path:progress:)`

## tasks

### Task 1: Add DatabaseManager CRUD methods

**File to modify:** `Prunr/Database/DatabaseManager.swift`

**Requirements:**
- `createSnapshot() throws -> Snapshot` - Creates new snapshot with current date
- `addEntry(to snapshotId: Int64, path: String, sizeBytes: Int64) throws` - Inserts single entry
- `addEntries(to snapshotId: Int64, entries: [FolderSize]) throws` - Batch inserts for performance
- `fetchAllSnapshots() throws -> [Snapshot]` - Returns all snapshots ordered by createdAt desc
- `fetchEntries(for snapshotId: Int64) throws -> [SnapshotEntry]` - Returns entries for a snapshot
- Use `inTransaction` for batch inserts

**Verification:**
- Build compiles: `xcodebuild -scheme Prunr -sdk macos build`
- Manual: Can create snapshot, add entries, fetch them back

---

### Task 2: Create ScanService orchestrator

**File to create:** `Prunr/Services/ScanService.swift`

**Requirements:**
- Singleton: `ScanService.shared`
- Method: `func scan(path: String, progress: ProgressHandler?) async throws -> Snapshot`
- Progress type: `typealias ProgressHandler = (String, Int) -> Void` (path, foldersScanned)
- Flow:
  1. Create new Snapshot via DatabaseManager
  2. Scan directory using DiskScanner
  3. Batch insert entries to DatabaseManager
  4. Return the completed Snapshot
- Errors propagate to caller for UI handling

**Verification:**
- Build compiles
- Manual: Running `await ScanService.shared.scan(path:progress:)` stores data in database
- Database contains Snapshot and SnapshotEntry rows after scan
