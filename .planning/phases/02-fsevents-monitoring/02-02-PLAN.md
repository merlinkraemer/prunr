  <action>Create `actor FSEventsWatcher` with:
  1. `private var stream: FSEventStream?` reference (using FSEventStream from CoreServices)
  2. `private var debounceTask: Task<Void, Never>?` for debounce timing
  3. `private let debounceInterval: TimeInterval = 3.0` (3 seconds default)
  4. `private var onChange: ((Set<URL>) -> Void)?` callback for changed paths
  5. `init(pathsToWatch: [URL], debounceInterval: TimeInterval = 3.0)` initializer
  6. `func start()` method that:
     - Creates FSEventStream using FSEventStreamCreate() with pathsToWatch
     - Sets callback to collect changed event paths
     - Starts stream with FSEventStreamStart()
     - Schedules stream on run loop with FSEventStreamScheduleWithRunLoop()
  7. `func stop()` method that stops and invalidates the stream
  8. Private `func handleEventPaths(_ paths: Set<URL>)` that debounces by:
     - Canceling existing debounce task
     - Creating new Task that sleeps for debounceInterval, then calls onChange

  Use `import CoreServices` for FSEvents APIs. FSEventStreamCreate signature:
  ```
  FSEventStreamCreate(allocator, callback, context, pathsToWatch, sinceWhen, latency, flags)
  ```
  Pass `kFSEventStreamEventIdSinceNow` for sinceNow, and 0.5 for latency (seconds).

  Callback signature: uses `FSEventStreamCallback` - capture changed paths from event array.</action>
  <verify>Build succeeds, FSEventsWatcher compiles without errors, FSEventStreamCreate is imported from CoreServices</verify>
  <done>FSEventsWatcher actor created with FSEventStream management, 3-second debounce, callback support</done>
