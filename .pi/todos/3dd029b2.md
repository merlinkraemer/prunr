## GitHub Issue #13
**Label:** improvement

In the drill-down file view (subcategory → files), the list of grown items currently shows all matching files without an upper bound. For large subcategories this creates an unwieldy scroll and overwhelms the user.

### Desired behavior
- Show at most 10 grown items by default.
- If more than 10 exist, display a "Show more" button / expandable section below the first 10.
- Clicking "Show more" reveals the remainder (or the next N batch).
- The existing "load more" pagination for non-growth files should remain unchanged.

### Scope
- `CategoryGrowthListView.fileListView` / `SubcategoryGroup` presentation layer.
- Growth contributors (`visibleGrowthContributors`) vs regular file rows (`nonGrowthFiles`).

### Acceptance criteria
- [ ] Drill-down file list never renders more than 10 growth-contributor rows initially.
- [ ] A clear affordance (button / row) appears when growth contributors > 10.
- [ ] Expanding does not trigger a new DB query; it only reveals already-fetched data.
- [ ] Existing "load more" pagination for total file count remains intact.
